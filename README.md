# 🚀 Dot Digital Group - Teste Prático

Projeto desenvolvido como parte de um teste técnico para vaga de desenvolvimento Front-End.

---

## 📌 Sobre o Projeto

Este projeto consiste na criação de uma interface web responsiva com diferentes seções, incluindo:

* Hero section
* Vídeo incorporado
* Carrossel de imagens
* Área de conteúdo (texto + imagem)
* Áudio player
* FAQ interativo (accordion)
* Atividades (discursiva e objetiva)
* Sistema de comentários
* Footer

O foco foi construir uma aplicação **moderna, organizada e responsiva**, aplicando boas práticas de HTML, CSS e JavaScript.

---

## ▶️ Como rodar o projeto

1. Clone o repositório:

```bash
git clone https://github.com/LucasFerreira-Dev/dot-digital-group.git
```

2. Acesse a pasta:

```bash
cd dot-digital-group
```

3. Abra o arquivo `index.html` no navegador:

* Clique duas vezes no arquivo
  ou
* Use uma extensão como **Live Server** no VS Code

## 🧠 Decisões Técnicas

### 🔹 HTML

* Estrutura semântica organizada
* Separação clara de seções (hero, main, footer)
* Uso de classes reutilizáveis

### 🔹 CSS

* Arquivo único (`global.css`) para centralizar estilos
* Uso de **Flexbox** para layout
* Responsividade com **Media Queries**
* Padronização de espaçamentos e tamanhos
* Organização por seções (hero, main, components, etc.)

### 🔹 JavaScript

* Manipulação de DOM para:

  * Accordion (FAQ)
  * Seleção de opções (atividade objetiva)
  * Interações de botões
* Código simples e direto, focado em legibilidade
* Persistência de dados com **sessionStorage** (restaura atividades ao recarregar)

### 🔹 Organização

* Separação por pastas:

  * `css/`
  * `js/`
  * `assets/` (imagens, ícones, áudio)

## 💬 Considerações Finais

Gostaria de destacar minha satisfação em participar deste teste técnico, mesmo atuando habitualmente com React + TypeScript, o que tornou o desafio ainda mais enriquecedor ao revisitar e estudar novamente os fundamentos do HTML e JS puros.

Foi uma ótima oportunidade para demonstrar minhas habilidades em desenvolvimento front-end, organização de código e atenção aos detalhes na construção de interfaces.

Estou sempre buscando evoluir e aplicar boas práticas no desenvolvimento, e ficarei muito feliz em contribuir com a equipe.

Infelizmente não separei o CSS em múltiplos arquivos por subestimar o tamanho do projeto, mas reconheço que não foi a melhor decisão

Vale ressaltar que, por conta do prazo de 3 dias, optei por priorizar a entrega funcional em detrimento de comentários no código — o que não é meu costume habitual
